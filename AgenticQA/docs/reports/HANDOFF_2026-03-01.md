### 2) Client Deployment Preflight Check (new module)
**File:** `src/agenticqa/client_preflight.py`

7-probe validation system that checks if a target repo is ready for AgenticQA execution:

| Probe | Checks | Fail/Warn |
|-------|--------|-----------|
| `git_repo` | Is it a git repo? | fail |
| `git_config` | user.name + user.email set? | fail |
| `python_tooling` | Python, pip, pytest available? | fail |
| `node_tooling` | If package.json: node/npm available? | warn |
| `linter_availability` | flake8/eslint per detected language? | warn |
| `path_sanitization` | Repo path passes sanitize_repo_path()? | fail |
| `import_chain` | agenticqa.workflow_requests loads? | fail |

CLI: `python -m agenticqa.client_preflight --repo . [--json] [--fail-on-warning]`
Exit codes: 0=healthy, 1=critical fail, 2=warnings only

**Tests:** 17 unit tests in `tests/test_client_preflight.py` ŌĆö all passing

### 4) PromptWorkflowStore Thread-Safety Fix (production bug)
**File:** `src/agenticqa/workflow_requests.py`

The concurrent worktree test exposed that most DB-accessing methods were missing `self._lock` acquisition. Only `atomic_pickup()` had it. This caused `sqlite3.InterfaceError: bad parameter or other API misuse` under concurrent worker load.

**Methods fixed (added `with self._lock:`):**
- `_transition()` ŌĆö used by complete_request, fail_request, start_request, etc.
- `get_request()`
- `get_queue_status()`
- `set_worktree_path()`
- `list_requests()`
- `get_next_queued_request()`
