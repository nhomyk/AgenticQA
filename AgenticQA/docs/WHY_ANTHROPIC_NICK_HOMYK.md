The honest answer is that I've already been working here, I just haven't been on the payroll.

AgenticQA runs on Claude. Every agent in the system, from the one that generates targeted tests for uncovered code paths to the one that runs adversarial bypass attempts against safety filters, calls the Anthropic API. I chose Claude not because it was available but because it was better for this specific use case: the reasoning is more reliable, the instruction-following is more consistent, and the outputs are easier to validate programmatically. Those things matter when agents are making decisions about production code.

But the part that feels most distinctly Anthropic is the thing I built that I didn't expect to care about as much as I do. The Constitutional Gate converts written policies into executable rules that govern every agent action in the system. When the Red Team agent discovers a new bypass pattern, it generates a proposal for human review rather than patching its own constraints. The rule that prevents self-modification of constitutional logic is called T1-005 in the codebase. I wrote it because it felt obviously correct: a system that can edit its own safety rules is a system you can't trust. That's not a principle I learned from a paper. It's one I arrived at independently while building, and then recognized it as exactly what Anthropic has been arguing publicly for years.

That alignment matters to me. I've spent most of my career in QA, which means I've spent most of my career thinking about what happens when systems fail in ways their designers didn't anticipate. The work Anthropic is doing on interpretability and alignment is the most serious attempt I've seen to apply that same discipline to AI systems at a scale where it really matters. I want to be part of an organization that treats that problem as foundational rather than a feature to add later.

The GRC role specifically is interesting because it's the same problem applied internally: how do you govern a fast-moving AI company in a way that actually scales? I've been building the answer to that question for software teams. I'd like to help build it for Anthropic.
