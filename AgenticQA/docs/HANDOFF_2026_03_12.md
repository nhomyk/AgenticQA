## What Was Done This Session

### Security Hardening Pass (commit `21cf6e8`)
22 vulnerabilities fixed across the Python codebase:

| Category | Fix |
|----------|-----|
| SQL column injection | Whitelist-based column guard in `workflow_requests.py` ‚Äî `_ALLOWED_COLUMNS` frozenset rejects unknown column names |
| SQL ORDER BY injection | Replaced dynamic `order_clause` string with dict lookup in `observability.py` |
| XXE | `plugin_onboarding.py` now uses `defusedxml.ElementTree` with stdlib fallback; `defusedxml>=0.7.1` added to `install_requires` in `setup.py` |
| Weak hashing | MD5 ‚Üí SHA-256 in `rag/vector_store.py` and `rag/embeddings.py`; SHA-1 ‚Üí SHA-256 in `data_store/developer_profile.py` |
| LLM code execution | `_check_llm_generated_code_safety()` AST scanner added to `agents.py` ‚Äî blocks eval, exec, os.system, rmtree, socket, etc. before `compile()` runs in `_attempt_test_repair()` and `_generate_tests_for_gap()` |
| Temp file permissions | `os.chmod(tmp_path, 0o600)` after NamedTemporaryFile in both LLM test generation paths |
| Token entropy | `secrets.token_hex(8)` ‚Üí `secrets.token_urlsafe(32)` in `destructive_action_interceptor.py` |
| Race condition (TOCTOU) | `approve()` now uses atomic `pop()` + moves entry to `self._approved` dict; `is_approved()` checks `_approved` first ‚Äî prevents double-approval exploit |
| Hardcoded credentials | Neo4j password default removed from `delegation_store.py` and `temporal_graph.py`; warns to stderr when `NEO4J_PASSWORD` env var unset |
| Path traversal info leak | `subprocess_runner.py` strips full path from error messages, only returns basename |
