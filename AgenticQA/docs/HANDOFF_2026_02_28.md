## What Was Built Today

### 1. Full Client Demo Script — `run_demo.py`

An 8-step end-to-end demo that runs the complete ADLC cycle without requiring an API key. Designed for live demos and CI validation.

**What it does:**
1. Creates a fresh temp repo with a minimal FastAPI backend (zero tests, zero frontend)
2. Runs 5-phase onboarding (architecture + security + coverage + test gen + baseline)
3. Submits a feature description via `POST /api/demo/submit`
4. Generates a Streamlit UI component (stub or real Claude Haiku if key present)
5. Runs architecture scan → security scan → UI tests
6. Self-heals test failures autonomously (up to 2 retries)
7. Records coverage delta
8. Issues SHIP IT / REVIEW REQUIRED verdict

**Key behaviors:**
- No API key: uses `DEMO_UI_CODE` stub. With `ANTHROPIC_API_KEY`: calls Claude Haiku
- `DEMO_REPO` env var lets you reuse an existing repo (skip creation)
- Dynamic gap analysis — warns only when actual coverage < 80%
- ANSI-safe verdict box with fixed-width padding
- Filters test files from architecture scan display

**Usage:**
```bash
python run_demo.py                              # stub mode, fresh repo
ANTHROPIC_API_KEY=sk-ant-... python run_demo.py  # real LLM mode
DEMO_REPO=/tmp/existing-repo python run_demo.py  # skip repo creation
```

### 2. Bug Fix — FrontendTestRunner `NO_TESTS_COLLECTED`

**Root cause:** `generated.test_command` was `"python -m pytest --tb=short -q"`. The runner appended `["--tb=short", "-q", "--no-header"]` — doubling `-q` into `-qq`, which suppresses the pytest summary line entirely. `_parse_pytest` parsed 0 results and returned `NO_TESTS_COLLECTED`.

**Fix** ([frontend_test_runner.py:218-230](../src/agenticqa/testing/frontend_test_runner.py#L218)):
- Strip all flags from `test_command`, keep only the interpreter prefix (`["python", "-m", "pytest"]`)
- Append `[test_path, "--tb=short", "-v", "--no-header"]` with `-v` (verbose) instead of `-q`
- `-v` produces `PASSED test_name` lines that `_parse_pytest` can parse

**Result:** 3/3 tests passing on first run.

### 4. `POST /api/demo/submit` Endpoint

Public endpoint — no auth required. Drives the landing page pipeline.

**Flow:**
```
POST /api/demo/submit { description, repo_path }
  → ArchitectureScanner.scan()
  → coverage before
  → stub UI generation → written to ui/feature.py
  → SecurityScanner (12-category)
  → FrontendTestRunner.run()
  → coverage after
  → { verdict, elapsed_s, security, tests, coverage }
```

**Auth note:** `/` and `/api/demo/submit` added to `_AUTH_SKIP_PATHS` in `api_middleware.py` so `BearerTokenMiddleware` does not block the landing page.
