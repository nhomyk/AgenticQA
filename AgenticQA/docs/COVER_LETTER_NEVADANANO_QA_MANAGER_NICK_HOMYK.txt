I've spent the better part of my career building quality programs at IoT companies from early stages, and the QA Manager role at NevadaNano is exactly the kind of work I want to be doing.

I joined when there was no formal QA function, and I built it: the automation framework, the API test coverage, the CI pipeline integration, the defect triage process, the release quality standards. I've since stepped back into a deep technical role by choice, because building and owning automation at the system level is where I do my best work.

That experience maps directly to what you're looking for. IoT platforms have a class of QA problems that generic SaaS doesn't: device-cloud data fidelity, async event flows, sensor-driven state transitions, multi-environment parity between hardware and simulated inputs. I've worked through those problems hands-on. I understand why a flaky test on a time-series API is a different problem than a flaky UI test, and I know how to build automation that handles both without becoming a maintenance burden.

On the automation side: I've been building with Python throughout my career and recently completed AgenticQA, an open-source multi-agent quality platform that integrates with GitHub Actions, runs automated compliance checks, attributes violations to developers via git blame, and learns which patterns are fixable versus structural across runs. It's a more complex system than most QA tooling I've worked with, and building it sharpened my thinking on test architecture, pipeline integration, and what scalable quality infrastructure actually looks like.

What draws me specifically to NevadaNano is the problem space. Gas detection and safety monitoring have a reliability bar that most software doesn't. A missed alert or a false negative in a safety-critical IoT platform isn't a UX problem; it has real consequences. That's the kind of quality work that requires genuine rigor, not just coverage metrics, and it's the environment where I've always done my best work.

I'd welcome the chance to talk about the role.
